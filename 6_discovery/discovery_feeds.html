This article covers the ways in which Feeds can be built, managed, and used.

A Feed is a set of rules that serves additional, suggested videos to viewers within a viewing session. With Feeds, viewers are empowered to discover and enjoy content they might not otherwise have found.

A publisher sets the business rules for a Feed, and when a valid JW Player Media ID (video) from that publisher's library is keyed into the Feed, it returns other videos from that library that best match the rules specified. The resulting set of videos is determined by the proprietary algorithms of JW Platform's Data-Driven Recommendations system.

Feeds are flexible, and can be used in several different ways in conjunction with video embeds: You can implement a Feed alongside a player (e.g. in a sidebar or below-player display), or even on top of a player (in a custom widget). Ultimately, a Feed is accessed via API endpoint, though Feed rules can be adjusted via the JW Player Dashboard

The Recommendations that a Feed delivers are powered by JW Platform's Data-Driven Recommendations system. If you do not have Data-Driven Recommendations or Feeds and would like to, please contact us.

 

Types of Data-Driven Feeds

Currently, there is one type of Feed: the 'Similar' Feed. This type of Feed will determine which videos are most similar to the source video based on that video's title & description metadata. By analyzing the keywords within the title & description metadata, our algorithms determine which other videos within the configured rule set (see below) will be most relevant.

In the future, we will be working on new types of Feeds, including one that will learn about your individual viewers and what they like to watch.

Configuration — 'Similar' Feeds

Similar Feeds use tag filter rules to give you a bit more control in limiting the set of recommended videos returned as 'similar'.

Filter by Tag options include:

  • 'All Content' — No tag filtration will occur and any video from your content library could potentially be considered 'available' for this feed (obviously determined by the designated Media ID).
  • 'Content with ALL of the below tags' — Only videos tagged with every one of the tags shown will be considered as 'available' to match for this feed. (This is the most restrictive case and could result in a very small matching video set).
  • 'Content with ANY of the below tags' — Any video with at least one of the tags shown will be considered 'available' for this feed. This provides a wider set of content, but still allows for some tag-based control.

You can also specify the maximum number of items that a feed will return (an integer up to 99).

Note: the Feed's maximum number of items is also limited by (a) the number of items in your library, that (b) match the tag rules you defined above. Add more content or adjust the rules to increase the number of 'available' videos.

Add Metadata to a Feed for SEO optimization, and to more easily search for the Feed within the dashboard or using the API.

Custom Parameters are key + value pairs that can be used to manage feeds.

 

Previewing a Feed

You'll want to preview your feed before implementation, to ensure that its algorithmic resuls behave as you would expect. This is especially true when first configuring a Feed or updating a Feed's rules. The 'preview' tool in the top right corner of the page accepts a source video (Media ID) and returns recommendations based on that video.



Enter a Media ID, then click 'Preview'.

In Feed Preview, you'll see the results for the Media ID (video) you entered, based on both the Feed's rules as well as other content from your library.

Remember, recommendations generated by a Feed (including by a Feed Preview) are specific to the moment in time in which the Feed was loaded. A different viewer may see slightly different results at another moment in time, based on the always-updating JW Platform Recommendations intelligence.

You might also want to learn more about how Data-Driven Recommendations work, or contact us to get started using Data-Driven Recommendations if you don't have them already.

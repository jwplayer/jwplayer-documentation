The JW Player product & engineering teams have been hard at work preparing a brand new Analytics experience for our publishers. When we started this project, we reached out to our users and got some amazing feedback about what could be better. And we heard you! So finally, over the next few months, we are rolling out some exciting improvements.

The first step to building out our new Analytics product was to strengthen our publishers’ data. We built a process by which any content registered in our system (via the dashboard or the API) is automatically assigned a unique Media ID (which is then included in the dashboard-provided embed code). Current Platform customers will be familiar with the concept — Media ID is just another name for Video Key.
 

What Do I Need to Know?

Using Media IDs allows us to accurately attribute all data points to a corresponding piece of content. One improvement in particular: Metrics from stolen player keys will no longer pollute Content and Audience Analytics data.

Plus, Media IDs set a powerful foundation for the other Analytics products and features we’ll release at the same time, like true video-level metrics.

 

Who Will Be Affected?

If you already use our dashboard to add content and generate embed code, no action needs to be taken. After the switch, you will only see backfilled data from August 1st — you may want to export your data prior to the October switch.

If you self-host, or use other methods of embedding content in your players, and want to have access to future Analytics product releases, please make sure that all of your pieces of content have a valid, JW Player-generated, Media ID by following the steps below.

 

How to Apply Media IDs

Note: If you already use the dashboard to add content and generate embed code, your setup work is already done.

  1. Get Media IDs for your content via the Dashboard or the API
    • Dashboard: Add externally-hosted URLs or upload your content via the dashboard, and find the Media ID on each piece of content’s detail page
      • Go to your Content section, and select “upload” or “add via URL” on the big action button on the top left
    • API: The /videos/create method supports externally-hosted URLs
      • Be sure to define sourcetype as 'url' and to also include your sourceurl

  2. From now on, embed all NEW content using Media IDs
    • Dashboard: Once your content is added, click “embed” on a video from the detail page or list page. (Publish Mode generates single-line embed codes that contain a reference to your content’s Media ID.)
    • API: Include the Media ID parameter in your player embeds, as in the below code example 
      playlist:[{
file:"",
mediaid:"eXaMPlE1"
}]

  3. To ensure analytics are returned for EXISTING content embeds:

    This will make sure that after the switch you will see backfilled data from the time you added Media IDs to these embeds.

    1. Replace those embeds with dashboard (single-line) embed code, or
    2. Make sure you add the above Media ID parameter into each of your existing embeds.

Quick FAQs

I already use Media IDs on my content. What should I do to prepare for the October switch?

  • If you require access to data from before August 1st (the date to which we will backfill data after the switch in October), you’ll want to export your data via dashboard CSV download or other methods. (This option is available in both the older dashboard as well as the new, JW7 dashboard).
  • Otherwise — you are already set up, and no further action is needed.

 

What is a Media ID?

  • A Media ID is an 8-character, alphanumeric string that is tied to one specific media object (like a video file). This ID is generated by our platform to differentiate between unique media objects.

 

How do I make sure my content is associated with valid Media IDs?

  • If you already use the dashboard to publish, no extra work is required.
  • If you publish using other methods, follow the instructions in “How to Apply Media IDs,” above.

 

Why is a valid Media ID now required for Analytics?

  • JW Player is releasing a few brand-new data & analytics products. Switching to Media-ID-filtered analytics was the first step to making sure these products provide you with valuable and actionable data.

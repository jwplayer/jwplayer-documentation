Here is an example of a setup of JW Player, presenting a single RTMP stream. By default, the video starts from the beginning. However, if there is a time offset hash in the URL, the video will start from that time.

Click here to reload the page with a time offset of 60 seconds. The link simply appends #t=60 to the URL.

 

Note that this functionality works for streaming formats (HLS, RTMP) and in HTML5 mode, but not in Flash mode for progressive (MP4, AAC).

This feature is built using the JW Player JavaScript API, with just a few lines of JavaScript code.

Embed code

As always, ensure you have uploaded a copy of JW Player and included the script in the <head> of your page, together with your player key. See the Quick Start Guide for more info. Next, place the following embed code at the location you want the player to appear:

<div id="myElement"></div>

<script>
  var playerInstance = jwplayer("myElement");
  playerInstance.setup({
    image: "/uploads/myPoster.jpg",
    file: "rtmp://example.com/vod/myStream.mp4"
  });
</script>

Second, add the JavaScript to implement the seek-from-offset:

<script>
  playerInstance.on('ready', function() { 
    if(window.location.hash) {
      var offset = window.location.hash.substr(3);
      playerInstance.seek(offset);
    }
  });
</script>

This script reads the offset from the browser hash and then uses playerInstance.seek() to seek to that offset. Enhancements can be made e.g. to retrieving the offset, or by not having the player autostart (by listening to onPlay() instead of onReady()). See Using the JavaScript API for more info.
